# Анализ результатов сравнения алгоритмов сортировки

Анализ производительности классического **Merge Sort** и гибридного **Merge+Insertion Sort**, где происходит переключение на сортировку вставками при достижении малого размера подмассива.


### Параметры тестирования
- Размеры массивов: 500–100000 (шаг 100)
- Threshold: 5, 10, 15, 20, 30, 50
- Типы массивов: случайные, обратно отсортированные, почти отсортированные
- 3 прогона для усреднения
- Метрика: микросекунды

## Основные результаты

### 1. Случайные массивы
- Лучший threshold: 15
- Ускорение: 8–15%
- Деградация после threshold > 30

### 2. Обратно отсортированные массивы
- Лучший threshold: 8
- Ускорение: 2–5%
- Деградация после threshold > 20

### 3. Почти отсортированные массивы
- Лучший threshold: 20
- Ускорение: 20–35%
- Широкий рабочий диапазон threshold: 10–30

## Анализ порогов

| Тип данных | Оптимальный threshold |
|-----------|------------------------|
| Случайные | 15 |
| Обратно отсортированные | 8 |
| Почти отсортированные | 20 |

## Улучшение производительности

| Тип данных | Улучшение |
|-----------|-----------|
| Случайные | ~12% |
| Обратно отсортированные | ~4% |
| Почти отсортированные | ~28% |


## Заключение

Гибридный Merge+Insertion Sort показывает стабильное ускорение до 35% при правильном threshold. Наиболее универсальное значение — 15.
